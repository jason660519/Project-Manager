# OpenClaw Plugin Operations

> Status: Active  
> Last updated: 2026-05-18  
> Audience: AI engineers and maintainers

---

## English Version

## 1. Purpose

Project Manager treats OpenClaw as a project-scoped external runtime. The app owns the wrapper scripts, plugin toggle, and runtime state location. OpenClaw owns its own source tree and dependency graph.

This keeps Project Manager decoupled from OpenClaw updates: OpenClaw can be updated or rolled back without importing its source code into the Next.js app, changing Project Manager dependencies, or letting OpenClaw runtime state leak into global `~/.openclaw` paths.

## 2. Boundaries

| Area | Path | Owner |
| --- | --- | --- |
| OpenClaw source checkout | `.project-manager/vendor/openclaw/` | OpenClaw upstream |
| Project wrapper scripts | `scripts/openclaw.sh`, `scripts/install-openclaw.sh`, `scripts/update-openclaw.sh`, `scripts/rollback-openclaw.sh` | Project Manager |
| Project runtime state | `.project-manager/openclaw/state/` | Project Manager local runtime |
| OpenClaw workspace | `.project-manager/openclaw/workspace/` | Project Manager local runtime |
| Project CLI wrapper | `.project-manager/bin/openclaw` | Generated by install script |
| Plugin toggle metadata | `lib/storage/plugins.ts`, `app/ui/views/PluginsView.tsx` | Project Manager |

Do not place OpenClaw source under `app/` or a top-level source directory. Next.js scans `app/**/route.ts` as application routes, and Turbopack file tracing can chase top-level vendored virtualenvs or package trees during builds.

## 3. Install

Run:

```bash
npm run openclaw:install
```

The installer:

- clones OpenClaw into `.project-manager/vendor/openclaw` if the checkout is missing;
- installs OpenClaw dependencies in `.project-manager/vendor/openclaw`;
- builds OpenClaw and its control UI;
- creates `.project-manager/openclaw/state/openclaw.json`;
- creates `.project-manager/openclaw/state/.env` with a project-scoped gateway token;
- creates `.project-manager/bin/openclaw`.

The generated `.env`, config, workspace, and manifest are intentionally ignored by git through `.project-manager/`.

Optional install inputs:

| Environment variable | Default | Purpose |
| :-- | :-- | :-- |
| `PM_OPENCLAW_REPO_URL` | `https://github.com/openclaw/openclaw.git` | Source repository to clone when missing. |
| `PM_OPENCLAW_REF` | unset | Specific branch, tag, or commit to checkout during install. |
| `PM_OPENCLAW_AUTO_UPDATE` | `0` | When set to `1`, fetch and checkout `origin/main` during install before building. |

## 4. Runtime Commands

```bash
npm run openclaw -- --version
npm run openclaw:doctor
npm run openclaw:gateway
npm run openclaw:dashboard
```

Project Manager always launches OpenClaw through `scripts/openclaw.sh`. Do not use a global `openclaw` binary for Project Manager tasks because it may read or write global user state.

Project Manager reserves gateway port `18790` for OpenClaw so it does not collide with a global OpenClaw install that commonly uses `18789`.

The wrapper exports these project-scoped paths:

- `OPENCLAW_HOME=.project-manager/openclaw/home`
- `OPENCLAW_STATE_DIR=.project-manager/openclaw/state`
- `OPENCLAW_WORKSPACE_DIR=.project-manager/openclaw/workspace`
- `OPENCLAW_CONFIG_PATH=.project-manager/openclaw/state/openclaw.json`

## 5. Plugin Toggle

OpenClaw is registered as a disabled CLI plugin by default. A user can enable it from the Plugins view. Once enabled, Project Manager exposes it as an agent adapter using:

```text
.project-manager/bin/openclaw agent --message {prompt}
```

Keep OpenClaw disabled by default. Installation and availability are separate from user intent to use it.

## 6. Update

To update to the upstream default branch:

```bash
npm run openclaw:update
```

To update to a specific tag, branch, or commit:

```bash
npm run openclaw:update -- v2026.5.17
npm run openclaw:update -- bef3356375
```

The update script records the previous and current refs in `.project-manager/openclaw/manifest.json`.

If install or build fails during update, the script attempts to restore the previous source ref before exiting.

## 7. Rollback

To roll back to the previous recorded ref:

```bash
npm run openclaw:rollback
```

To roll back to an explicit ref:

```bash
npm run openclaw:rollback -- bef3356375
```

Rollback rebuilds OpenClaw after checkout so the CLI and gateway match the selected source ref.

## 8. Verification

After install, update, or rollback, run:

```bash
npm run openclaw -- --version
npm run openclaw:doctor
npm run typecheck
npm run build
```

If `npm run build` starts compiling OpenClaw files, check that OpenClaw source is outside `app/` and top-level source directories, and excluded from Project Manager TypeScript scanning.

## 中文版本

## 1. 目的

Project Manager 將 OpenClaw 視為專案 scope 內的外部 runtime。本專案只負責 wrapper scripts、plugin toggle 與 runtime state 位置；OpenClaw 的 source tree 與 dependencies 由 OpenClaw 自己管理。

這樣做可以讓 Project Manager 與 OpenClaw 解耦：OpenClaw 可以獨立更新或回滾，不需要把 OpenClaw source import 進 Next.js app，不污染 Project Manager dependencies，也不把 runtime state 寫到全域 `~/.openclaw`。

## 2. 邊界

| 區域 | 路徑 | 負責方 |
| --- | --- | --- |
| OpenClaw source checkout | `.project-manager/vendor/openclaw/` | OpenClaw upstream |
| Project wrapper scripts | `scripts/openclaw.sh`, `scripts/install-openclaw.sh`, `scripts/update-openclaw.sh`, `scripts/rollback-openclaw.sh` | Project Manager |
| Project runtime state | `.project-manager/openclaw/state/` | Project Manager local runtime |
| OpenClaw workspace | `.project-manager/openclaw/workspace/` | Project Manager local runtime |
| Project CLI wrapper | `.project-manager/bin/openclaw` | install script 產生 |
| Plugin toggle metadata | `lib/storage/plugins.ts`, `app/ui/views/PluginsView.tsx` | Project Manager |

不要把 OpenClaw source 放在 `app/` 或頂層 source 目錄底下。Next.js 會把 `app/**/route.ts` 當成本專案路由掃描，Turbopack file tracing 也可能在 build 時追到頂層 vendored virtualenv 或 package tree。

## 3. 安裝

執行：

```bash
npm run openclaw:install
```

installer 會：

- 在 `.project-manager/vendor/openclaw` 安裝 OpenClaw dependencies；
- 若 checkout 不存在，自動 clone 到 `.project-manager/vendor/openclaw`；
- build OpenClaw 與 control UI；
- 建立 `.project-manager/openclaw/state/openclaw.json`；
- 建立 `.project-manager/openclaw/state/.env`，內含專案 scope 的 gateway token；
- 建立 `.project-manager/bin/openclaw`。

產生的 `.env`、config、workspace、manifest 都透過 `.project-manager/` 忽略，不進 git。

可選 install 參數：

| Environment variable | Default | 用途 |
| :-- | :-- | :-- |
| `PM_OPENCLAW_REPO_URL` | `https://github.com/openclaw/openclaw.git` | checkout 不存在時要 clone 的 source repository。 |
| `PM_OPENCLAW_REF` | unset | install 時指定 checkout 的 branch、tag 或 commit。 |
| `PM_OPENCLAW_AUTO_UPDATE` | `0` | 設為 `1` 時，install/build 前會 fetch 並 checkout `origin/main`。 |

## 4. Runtime Commands

```bash
npm run openclaw -- --version
npm run openclaw:doctor
npm run openclaw:gateway
npm run openclaw:dashboard
```

Project Manager 一律透過 `scripts/openclaw.sh` 啟動 OpenClaw。不要用全域 `openclaw` binary 操作 Project Manager，因為它可能讀寫全域 user state。

Project Manager 的 OpenClaw gateway 固定使用 `18790`，避免跟常見全域 OpenClaw install 使用的 `18789` 衝突。

wrapper 會指定這些 project-scoped paths：

- `OPENCLAW_HOME=.project-manager/openclaw/home`
- `OPENCLAW_STATE_DIR=.project-manager/openclaw/state`
- `OPENCLAW_WORKSPACE_DIR=.project-manager/openclaw/workspace`
- `OPENCLAW_CONFIG_PATH=.project-manager/openclaw/state/openclaw.json`

## 5. Plugin Toggle

OpenClaw 預設註冊為 disabled CLI plugin。使用者可在 Plugins view 自行啟用。啟用後，Project Manager 會用以下命令把它暴露為 agent adapter：

```text
.project-manager/bin/openclaw agent --message {prompt}
```

OpenClaw 必須維持預設 disabled。安裝完成只代表可用，不代表使用者已選擇使用。

## 6. 更新

更新到 upstream 預設 branch：

```bash
npm run openclaw:update
```

更新到指定 tag、branch 或 commit：

```bash
npm run openclaw:update -- v2026.5.17
npm run openclaw:update -- bef3356375
```

update script 會在 `.project-manager/openclaw/manifest.json` 記錄 previous 與 current refs。

如果 update 的 install/build 階段失敗，script 會嘗試 checkout 回 previous source ref 再退出。

## 7. 回滾

回滾到上一個記錄的 ref：

```bash
npm run openclaw:rollback
```

回滾到明確指定的 ref：

```bash
npm run openclaw:rollback -- bef3356375
```

rollback 會在 checkout 後重新 build OpenClaw，讓 CLI 與 gateway 對齊所選 source ref。

## 8. 驗證

安裝、更新、回滾後，執行：

```bash
npm run openclaw -- --version
npm run openclaw:doctor
npm run typecheck
npm run build
```

如果 `npm run build` 開始編譯 OpenClaw files，先檢查 OpenClaw source 是否仍在 `app/` 與頂層 source 目錄外，並且已排除於 Project Manager TypeScript 掃描之外。
