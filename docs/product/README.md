## 3. Rules

1. Product docs go under `docs/product/` with English kebab-case filenames.
2. Do not put product docs in the repo root or `docs/` root.
3. Technical implementation contracts belong in `docs/engineering/`.
4. Durable architecture decisions belong in `docs/architecture/`.
5. Deprecated product docs move to `docs/archive/archived-YYYYMMDD-name.md` with a short deprecation note.
